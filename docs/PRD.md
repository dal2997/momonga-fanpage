## 1. 제품 개요

### 1.1 한 줄 정의

> **momonga는 굿즈·수집품을 아름답게 기록하고, 원하는 사람과 거래할 수 있는 수집 트래커다.**

### 1.2 브랜드 방향

| 항목 | 내용 |
|------|------|
| **제품명** | momonga |
| **도메인** | momonga.app |
| **마스코트** | 모몽가 (작고 둥글고 귀여운 날다람쥐) |
| **슬로건** | *모으는 즐거움, 기록하는 설렘* |
| **톤** | 따뜻하고 감성적, 덕질 감성, 과하지 않게 귀엽게 |

### 1.3 포지셔닝 변화

| 현재 | 목표 |
|------|------|
| 특정 캐릭터(모몽가·코기뮹·짱구) 팬사이트 | 누구나 쓰는 범용 굿즈·수집 트래커 |
| 관리자 1인 수집품 공개 | 다수 유저의 수집품 관리 + 공개 + 거래 |
| 초대코드 기반 폐쇄형 | 단계적 개방 (초대 → 오픈 베타 → 퍼블릭) |

## 2. 문제 정의

### 2.1 타겟 유저

**Primary**: 굿즈·피규어·포카·한정판 아이템을 적극적으로 수집하는 2030 덕후

**Secondary**: 자신의 소장품을 정리하고 공유하고 싶은 라이트 컬렉터

### 2.2 현재 유저의 고통점

| # | 고통점 | 현재 해결 방식 | 문제 |
|---|--------|--------------|------|
| 1 | 뭘 샀는지 기억 안 남 | 메모장, 엑셀 | 이미지 없음, 보기 불편 |
| 2 | 내 수집품 가치를 모름 | 당근/번개장터 검색 | 매번 찾기 귀찮음 |
| 3 | 소장품 공유하기 어려움 | 인스타 피드, 트위터 | 업데이트 번거로움, 정리 안 됨 |
| 4 | 거래할 때 연락처 교환 불편 | DM | 스팸, 개인정보 노출 |
| 5 | 중복 구매 | 기억에 의존 | 돈 낭비 |

### 2.3 기회

- 국내 굿즈 시장 고속 성장 (아이돌 포카, 팝마트, 애니 피규어 등)
- 수집 전용 서비스가 없음 — Notion, 엑셀, 카페 게시판이 대체재
- 덕후 커뮤니티 내 "내 수집 현황 공유" 수요 높음

## 3. 제품 목표

### 3.1 단기 목표 (v1 오픈 베타, ~3개월)

- 유저 수: 가입자 **500명**, 활성 **200명** 이상
- 수집품 등록 수: **5,000개** 이상
- 오퍼(거래 요청) 발송 수: **500건** 이상
- 이탈 없는 온보딩 완료율: **70%** 이상

### 3.2 중기 목표 (v2, ~6개월)

- 오픈 회원가입 (초대코드 없이)
- 캐릭터/태그 기반 검색 기능
- 월간 활성 유저 **2,000명**

## 4. 핵심 기능 (MVP — 현재 구현 기준)

### 4.1 수집 관리

| 기능 | 설명 | 상태 |

| 드래그 정렬 | 카테고리 내 순서 변경 | ✅ 완료 |

| 기능 | 설명 | 상태 |
|------|------|------|

## 5. 로드맵 — 다음 단계

### Phase 2: 범용 트래커 확장 (v1.5)

#### 5.1 캐릭터 종속 제거

현재 구조는 "모몽가·코기뮹·짱구"라는 특정 캐릭터가 홈 화면에 고정되어 있음.
범용 트래커로 전환 시 홈을 **수집 현황 대시보드**로 교체.

- [ ] 홈 화면 → 내 수집 요약 대시보드 (총 아이템 수, 총 가치, 최근 추가)
- [ ] 캐릭터 탭 → 유저 지정 "테마" 또는 제거
- [ ] 온보딩 플로우 추가 (첫 방문 유저용)

#### 5.2 검색 & 태그

- [ ] 아이템 제목 검색 (탐색 + 내 수집)
- [ ] 시스템 태그: 캐릭터명, 시리즈명, 브랜드 (팝마트, 산리오, 디즈니 등)
- [ ] 태그 기반 필터링

#### 5.3 통계 강화

- [ ] 수집 총 가치 (원가 합산 / 중고 시세 합산)
- [ ] 월별 지출 추이 차트
- [ ] 카테고리별 분포 도넛 차트

#### 5.4 모바일 최적화

- [ ] 모달 → Bottom sheet 전환 (모바일)
- [ ] 빠른 추가 플로우 간소화
- [ ] PWA 설정 (홈 화면 추가)

### Phase 3: 오픈 & 소셜 (v2)

- [ ] 초대코드 없이 회원가입 가능
- [ ] 팔로우 / 위시리스트 공유
- [ ] OG 이미지 (소셜 공유 썸네일)
- [ ] 이메일 다이제스트 (주간 수집 요약)
- [ ] 아이템 단위 탐색 (유저 카드 → 아이템 검색)

## 7. 비즈니스 모델 (가설)

| 모델 | 설명 | 우선순위 |
|------|------|---------|
| **프리미엄** | 기본 무료, Pro 플랜 (무제한 아이템, 통계, 커스텀 URL) | 🔴 핵심 |
| **광고 없음** | 광고 없이 깨끗한 UX 유지 | ✅ 원칙 |
| **거래 수수료** | 오퍼 성사 시 소액 수수료 (장기) | 🟡 검토 |

### Pro 플랜 예상 혜택

- 아이템 무제한 (무료: 100개 제한)
- 수집 통계 대시보드
- 커스텀 핸들 URL 우선권
- 우선 고객 지원

## 8. 성공 지표 (KPI)

| 지표 | 측정 방법 | 목표 (3개월) |
|------|----------|------------|
| 가입자 수 | DB `profiles` count | 500명 |
| 월간 활성 유저 | 30일 내 로그인 | 200명 |

| 오퍼 발송 수 | DB `offers` count | 500건 |
| 거래 완료율 | completed / total offers | 20% 이상 |
| D7 리텐션 | 가입 후 7일 재방문 | 40% 이상 |
